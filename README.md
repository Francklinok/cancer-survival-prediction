## Overview

`ml_framework` is a **production-ready MLOps pipeline** for medical data analysis. It implements a complete **10-step workflow** with:

- Typed state management between steps
- Zero-leakage normalization (train/test split before fitting)
- Nested cross-validation for unbiased performance estimation
- Automatic Model Card generation (Google 2019 standard)
- SHAP explainability + clinical risk profiles
- Data drift monitoring (PSI, KS, Wasserstein)

The architecture follows industry patterns from **ZenML**, **Metaflow**, and **Kubeflow** — each pipeline step is an independent, testable class connected through a shared typed context object.

```python
# Hook system — attach monitoring without modifying the pipeline
def log_shape(ctx):
    print(f"  Shape after step: {ctx.df_work.shape}")

pipeline.add_hook("clean",   "after", log_shape)
pipeline.add_hook("outliers","after", log_shape)
pipeline.run("data.csv", target_column="Recurrence")
```

## Pipeline Architecture

```
PipelineContext      ← typed shared state (replaces raw dict)
       ↓
PipelineStep (ABC)   ← each step = autonomous class with requires / skip_if
       ↓
StepRegistry         ← step register + before/after hooks + ordering
       ↓
PipelineRunner       ← orchestrates without knowing business logic (+ retry)
       ↓
MedicalMLPipeline    ← public facade: configure, run, access results
```

### 10-Step Pipeline

| # | Step | Description |
|---|------|-------------|
| 1 | `ingest` | Load & validate data — CSV, Excel, Parquet, JSON |
| 2 | `clean` | Deduplication, near-zero-variance removal, type inference |
| 3 | `eda` | Distributions, target analysis, bivariate + diagnostic analysis |
| 4 | `missing` | MICE / KNN / MissForest — auto-skipped if < 0.1% NaN |
| 5 | `outliers` | IQR/Z-score/IsolationForest + winsorize/cap/impute treatment |
| 6 | `encode` | One-hot / target / frequency encoding |
| 7 | `normalize` | **Zero-leakage** — split BEFORE fitting transformers |
| 8 | `features` | Medical feature engineering + statistical selection (FDR/Bonferroni) |
| 9 | `train` | Nested CV + RandomizedSearchCV hyperparameter tuning |
| 10 | `evaluate` | Metrics + SHAP + feature importance + **Model Card** |

## Configuration

```python
from ml_framework.config.config import FrameworkConfig

config = FrameworkConfig()
config.model.test_size                    = 0.20
config.model.cv_folds                     = 5
config.model.models_to_test               = ["RandomForest", "GradientBoosting", "LogisticRegression"]
config.model.perform_hyperparameter_tuning = True
config.data.missing_strategy              = "mice"    # mice | knn | miss_forest | simple
config.data.outlier_method                = "iqr"     # iqr | zscore | isolation_forest
config.normalization.method               = "auto"    # auto | standard | minmax | robust

pipeline = MedicalMLPipeline(config=config)
pipeline.run("data.csv", target_column="Recurrence")
```

## Design Principles

| Principle | Implementation |
|-----------|---------------|
| **Zero data leakage** | Train/test split before fitting any transformer |
| **Modularity** | Every step is an independent class, usable standalone |
| **Typed state** | `PipelineContext` dataclass — no raw `dict` state |
| **Hooks system** | `add_hook("step", "before"\|"after", fn)` |
| **Auto-skip** | Steps skip when not needed (e.g. MissingStep if 0 NaN) |
| **Full traceability** | Execution times, errors, artifacts in `PipelineContext` |
